Beskriv din arbetsprocess

Beskriv i detalj hur din arbetsprocess för utveckling och publicering fungerar. Beskriv vad som är automatiserat och vilka verktyg du använt

Här skulle jag vilja skriva hur fantastisk min process varit, perfekt och genomtänkt in i detaljnivå. Sanningen är dock att ny kurs som vanligt är ett nytt språk för mig och processen har varit rörig där jag mest suttit och inte förstått något och plötsligt är uppgiften löst. Nedan är mitt försök att beskriva den.

Inledning

Jag inledde processen med att stirra in i skärmen och fundera på vad det är jag ska göra. Därefter gjorde jag det enda jag kan - jag skapade en hemsida med CSS och HTML för att ha grunden klar.

Efter det skapade jag tre branches "main", "development" och "prod" och en .gitignore fil. Jag skapade alltså i princip hela basen för hemsidan innan vilket jag insåg här att det var dumt. Jag var livrädd för alla termer och valde göra det enda trygga jag kunde - lägga in html och CSS. Pushade min git ignore fil till main men sedan undrade jag om jag gjorde rätt och pushade till development också. Sedan tänkte jag att jag kanske inte alltid ska göra först och tänka sedan så googlade massvis och frågade min vän som är utvecklare. Han sa att man brukar lägga .gitignore i rooten av projektet så den plockas upp i alla subfolders (han fick även förklara vad det betyder).

JavaScript

I nästa steg la jag till JavaScript. Jag valde att skapa en ny undersida för detta. Då jag jobbat väldigt lite i JavaScript så anpassade jag och la in en övning jag gjorde från ett bootcamp jag köpte under kursen i JavaScript under höstterminen där jag tränade if och if else satser

Publicera sida

Därefter gick jag till Git hub och valde att publicera min sida via Git hub pages. Valde då branch prod.

Katastrof

Började pyssla med Vite och följde guiden i modul 1. När allt blivit katastrof insåg jag mitt misstag vilket du som lärare redan insett säkert..... Jag lärde mig en väldigt hård och tråkig väg att man startar viteprojekt först. Så gjorde detta via terminalen och fick klistra in all min kod från tidigare projekt i korrekt mappar i min vite-mapp istället. Därefter fick jag börja om med git och github igen.

Saker jag lärt mig hitiills

  • Starta Vite-projekt först
  • Git ignore och liknande skapas automatiskt i vite du behöver inte ordna det manuellt
  • Kolla instruktionsfilmer och läs ordentligt först istället för ditt vanliga trial and error

Börjar på ny kula

Flyttade all kod till mitt-viteprojekt istället. Skapade mapp på Git hub och publicerade till render och valde branch prod där. Installerade plugin för fotooptimerare enligt Mattias video. Svor X antal gånger.
