Beskriv din arbetsprocess

Här skulle jag vilja skriva hur fantastisk min process varit, perfekt och genomtänkt in i detaljnivå. Sanningen är dock att ny kurs som vanligt är ett nytt språk för mig och processen har varit rörig där jag mest suttit och inte förstått något och plötsligt är uppgiften löst. Nedan är mitt försök att beskriva den. Kan tilläggas att första meningarna här skrevs innan kaoset - det blev ännu värre än jag hade kunnat föreställa mig.

Inledning

Jag inledde processen med att stirra in i skärmen och fundera på vad det är jag ska göra. Därefter gjorde jag det enda jag kan - jag skapade en hemsida med CSS och HTML för att ha grunden klar.

Efter det skapade jag tre branches "main", "development" och "prod" och en .gitignore fil. Pushade min git ignore fil till main men sedan undrade jag om jag gjorde rätt och pushade till development också. Sedan tänkte jag att jag kanske inte alltid ska göra först och tänka sedan så googlade massvis och frågade min vän som är utvecklare. Han sa att man brukar lägga .gitignore i rooten av projektet så den plockas upp i alla subfolders (han fick även förklara vad det betyder).

JavaScript

I nästa steg la jag till JavaScript. Jag valde att skapa en ny undersida för detta. Då jag jobbat väldigt lite i JavaScript så anpassade jag och la in en övning jag gjorde från ett bootcamp jag köpte under kursen i JavaScript under höstterminen där jag tränade if och if else satser

Publicera sida

Därefter gick jag till Git hub och valde att publicera min sida via Git hub pages. Valde då branch prod.

Katastrof

Började pyssla med Vite och följde guiden i modul 1. När allt blivit katastrof insåg jag mitt misstag vilket du som lärare säkert insåg första meningarna..... Jag lärde mig en väldigt hård och tråkig väg att man startar viteprojekt först. Så gjorde detta via terminalen och fick klistra in all min kod från tidigare projekt i korrekt mappar i min vite-mapp istället. Därefter fick jag börja om med git och github igen, eller ja hela processen. Har därför publicerat sida 1 och github projekt 1 i korrekt github repo under readme för att ni ska se alla commits från början. Dock i kaoset försvann hela min JavaScript sida bland annat - har ingen aning om vad som hände här men det var inte roligt i alla fall! Ni får analysera kaoset på första sidan och repot som en komedifilm och njuta av en rookies bekännelser..

Saker jag lärt mig hitiills

  • Starta Vite-projekt först
  • Git ignore och liknande skapas automatiskt i vite du behöver inte ordna det manuellt
  • Kolla instruktionsfilmer och läs ordentligt först istället för ditt vanliga trial and error

Börjar på ny kula

Flyttade all kod till mitt-viteprojekt istället. Skapade mapp på Git hub och publicerade till render och valde branch prod där. Installerade plugin för fotooptimerare enligt Mattias video. Svor X antal gånger.

Efter att ha testat att optimera bilderna via pluginet la jag in dessa i en grid och experimenterade mig fram till en okej layout

Valde dessvärre att inte justera bilderna i Photopea, spara i olika format samt infoga picture tag då mitt klanteri tidigare beskrivet kostade mig timmar.

Ny dag nya tag

Insåg att jag måste spara i olika format eftersom uppgiften kräver det. Sparade därför ner samtliga bilder med 1000px width och height auto i Webp och JPG. La därefter in i picture element. Optimerade därefter bilderna med mitt plugin som ordnades igår. Sparade med hjälp av det 87% och gick från 18.6mb till 2.3mb på bilderna.
